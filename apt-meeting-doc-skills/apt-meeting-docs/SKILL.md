# 입주자대표회의 문서 자동 생성 스킬

## 개요
3년치 과거 자료(안건·안건자료·회의록)를 아카이브로 내장하고, 이를 근거로:
1. **안건 → 안건자료** 자동 생성
2. **공고문 → 회의록 초안** 자동 생성

고정 정보: **삼부4단지 아파트 입주자대표회의** (대전 소재, 공동주택관리법 적용)

## 파일 구조
```
apt-meeting-docs/
├── SKILL.md
├── scripts/
│   ├── extract_hwpx.py      # HWPX 텍스트 추출
│   └── build_index.py       # 아카이브 인덱스(index.json) 생성
└── references/
    ├── formats.md           # 안건자료·회의록 출력 서식 템플릿
    └── archive/
        ├── index.json       # 전체 아카이브 인덱스 (검색 진입점)
        ├── agendas/         # 안건 및 안건자료 (연도별 .md)
        └── minutes/         # 회의록 (연도별 .md)
```

## 모드 판별 (가장 먼저 수행)
| 사용자 입력 | 모드 |
|---|---|
| 과거 자료 파일 다수 업로드 + "분류/정리/등록" | **모드 A: 아카이브 구축·갱신** |
| 안건 제목/주제 텍스트 + "안건자료" | **모드 B: 안건자료 생성** |
| 공고문(파일 또는 텍스트) + "회의록" | **모드 C: 회의록 초안 생성** |

모드 B·C 실행 전 반드시 `references/archive/index.json`을 읽는다.
index.json이 비어 있으면(항목 0건) 사용자에게 "아카이브가 비어 있어 일반 양식으로만 생성됩니다. 3년치 자료를 업로드하면 품질이 크게 올라갑니다"라고 안내하고, 일반 양식(`references/formats.md`)으로 진행한다.

## 모드 A: 아카이브 구축·갱신

### A-1. 파일 수집 및 텍스트 추출
업로드된 파일을 형식별로 추출:
- `.hwpx` → `python <스킬경로>/scripts/extract_hwpx.py "<파일>" "<출력.json>"` (실패 시 Preview/PrvText.txt fallback)
- `.docx` → pdf-reading/file-reading 스킬 관례대로 python-docx 또는 pandoc 사용
- `.pdf` → pdftotext 등으로 추출

### A-2. 문서 분류
각 문서를 아래 3종으로 분류 (파일명·내용 단서 활용):
- **안건(agenda)**: "제N안", "안건", "제안서" 포함, 단일 안건 구조
- **회의록(minutes)**: "회의록", "참석", "의결", "발언" 포함, 회의 진행 기록 구조
- **공고문(notice)**: "공고", "개최", 회의 일시·장소·안건 목록 구조 → 회의록의 메타데이터로 활용

### A-3. 표준 마크다운으로 정리 저장
스킬 폴더는 읽기 전용이므로 **먼저 `/home/claude/apt-meeting-docs/`로 스킬 전체를 복사**한 후 작업한다.

- 안건: `references/archive/agendas/{연도}/{회차}_{안건번호}_{제목슬러그}.md`
- 회의록: `references/archive/minutes/{연도}/{회차}_회의록.md`

각 파일 상단에 YAML 메타데이터 필수:
```yaml
---
type: agenda | minutes
year: 2025
session: 제7회        # 정기/임시 구분 포함 (예: 제7회 정기)
date: 2025-06-15
title: 아스콘 포장공사 사업자 선정의 건
category: 공사 | 용역 | 예산 | 규정 | 보고 | 기타
budget: 12,000,000원   # 없으면 생략
result: 원안 가결 | 수정 가결 | 부결 | 보류   # 회의록에서 확인된 경우
keywords: [아스콘, 포장, 주차장]
---
```

### A-4. 인덱스 재생성 및 재패키징
```bash
python /home/claude/apt-meeting-docs/scripts/build_index.py /home/claude/apt-meeting-docs/references/archive
cd /mnt/skills/examples/skill-creator && rm -f /home/claude/apt-meeting-docs.skill
python -m scripts.package_skill /home/claude/apt-meeting-docs
```
생성된 `.skill` 파일을 `present_files`로 전달하고, **재설치(Save skill)해야 아카이브가 반영됨**을 안내한다.

## 모드 B: 안건 → 안건자료 생성

### B-1. 유사 안건 검색
1. `references/archive/index.json` 로드
2. 입력 안건과 keywords·category·title 매칭으로 유사 안건 상위 2~3건 선정
3. 해당 `.md` 원문을 읽어 구조·문체·예산 규모·근거 법령 파악

### B-2. 안건자료 작성
`references/formats.md`의 **안건자료 서식**을 따르되, 유사 과거 안건의 실제 표현·항목 구성을 우선 반영:
- 제안이유 / 현황 및 문제점 / 세부 내용 / 소요예산 / 근거 규정(공동주택관리법·관리규약 조문) / 기대효과 및 향후계획
- 예산이 필요한 안건이면 과거 유사 공사·용역의 예산 규모를 참고치로 제시
- 사업자 선정 안건이면 「주택관리업자 및 사업자 선정지침」 관련 조항 명시
- 장기수선 관련이면 장기수선계획 조정(수시조정 포함) 필요 여부 언급

### B-3. 출력
- 기본: **docx** (docx 스킬의 SKILL.md를 읽고 그 절차대로 생성)
- 사용자가 hwpx 요청 시: `hwpx-agenda-converter` 스킬로 연계
- 확정 불가 정보(정확한 견적, 일자 등)는 `[  ]` 괄호로 비워두고 목록으로 안내
- `present_files`로 전달

## 모드 C: 공고문 → 회의록 초안 생성

### C-1. 공고문 파싱
공고문에서 추출: 회차(정기/임시), 일시, 장소, 안건 목록(번호·제목), 참석 대상

### C-2. 과거 회의록 패턴 로드
1. `index.json`에서 최근 회의록 2~3건의 `.md`를 읽어 **고정 구조** 파악 (개회 선언 문구, 성원 보고 방식, 안건 처리 서술 방식, 폐회 문구)
2. 공고문의 각 안건과 유사한 과거 안건의 의결 결과·논의 패턴 검색

### C-3. 회의록 초안 작성
`references/formats.md`의 **회의록 서식** + 과거 회의록의 실제 문체를 따라:
- 회의 개요(회차·일시·장소), 참석자 현황(성원 보고), 개회 선언
- 안건별 섹션: 제안 설명 요약 → 주요 질의·논의(과거 유사 안건 패턴 기반 예상 논의 포함) → 의결 결과 칸
- **채워야 할 항목은 명확히 표시**: `[참석: __명 / 재적: __명]`, `[찬성 __ / 반대 __ / 기권 __]`, `[의결 결과:      ]`
- 폐회 및 서명란

### C-4. 출력
docx로 생성 → `present_files` 전달. 초안임을 명시하고 회의 후 채울 항목 목록을 함께 안내.

## 주의사항
- 아카이브의 개인정보(동·호수, 개인 성명)는 생성 문서에 그대로 복사하지 않는다 (직책명만 사용: "회장", "감사", "○동 대표")
- 의결 결과는 절대 임의로 확정 기재하지 않는다 — 회의록 초안은 빈칸 처리
- 법령 조문 인용 시 조문 번호가 불확실하면 web_search로 현행 공동주택관리법 확인
- 날짜 표기: `2026. 07. 02.` 형식 (한국 공문서 관례)
